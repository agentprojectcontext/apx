Cada agente puede tener un archivo `memory.md` que APX inyecta en su system prompt en cada
llamado. Esta es la **memoria de largo plazo curada** del agente — un lugar para hechos duraderos
que siempre deben estar en contexto, no historial de conversación en bruto.

## Dónde vive

```text
~/.apx/projects/<apx_id>/agents/<slug>/memory.md
```

Este archivo es estado runtime local, fuera del repositorio. El repo commiteado conserva la
definición del agente en `.apc/agents/<slug>.md`; APX guarda la memoria bajo `~/.apx` para que
notas privadas, contexto del owner y detalles operativos no se filtren a repos públicos. El path
legacy `.apc/agents/<slug>/memory.md` se lee sólo como fallback de migración.

## Qué pertenece a la memoria

La memoria debe contener hechos estables y duraderos — cosas que el agente siempre debería saber:

- Las convenciones del proyecto ("usamos pnpm, nunca npm")
- Las preferencias del agente ("prefiere PR summaries cortos, pushea contra diffs vagos")
- Decisiones arquitectónicas clave ("la API es REST sobre HTTP, sin GraphQL")
- Responsabilidades en curso ("gestiona el pipeline de deploy a staging")

**Lo que NO pertenece acá:**

- Transcripts de conversaciones en bruto (inflan el prompt y se vuelven obsoletos)
- Secretos o API keys (la memoria runtime es local, pero credenciales igual no pertenecen al prompt)
- Contexto de tareas temporales (para eso usá sesiones)

<Aside type="caution">
  Nunca escribas secretos, tokens ni contraseñas en `memory.md`. Incluso memoria local de prompt
  puede copiarse, exportarse o enviarse a un modelo.
</Aside>

## Comandos

### Leer la memoria de un agente

```bash
apx memory reviewer                              # imprimir a stdout
apx memory reviewer --project my-app            # proyecto explícito
```

### Agregar una nota

```bash
apx memory reviewer --append "Prefiere PR summaries cortos"
apx memory reviewer --append "Usa locale es-AR para toda la salida"
```

La nota se agrega bajo un encabezado `## Recent context` (que se crea si no existe). Esta es
la forma recomendada de agregar un hecho puntual sin tocar el resto del archivo.

### Reemplazar la memoria

```bash
apx memory reviewer --replace < new-memory.md   # reemplazo completo desde stdin
```

Usá esto para una reescritura limpia — por ejemplo después de un pase de compactación, o para
importar masivamente un documento de memoria estructurado.

## Memoria y el system prompt

Cuando APX construye el system prompt para un agente (`apx exec <slug>` o cualquier llamado de
routine), el archivo de memoria se inyecta en la posición 5 del stack de prompts — después de
los campos de role/language y antes de las skills. Esto significa que el contenido de la memoria
siempre es visible para el modelo dentro de la ventana de contexto.

Para el sistema RAG cross-channel que recupera mensajes pasados semánticamente relevantes (no sólo
el archivo de memoria curado), consultá [Sistema de memoria](/apx/es/capabilities/memory-system/).
