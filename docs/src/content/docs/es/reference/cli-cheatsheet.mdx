## Bootstrap

| Comando | Qué hace |
|---|---|
| `apx init [path]` | Inicializa un proyecto APC (crea `AGENTS.md` + `.apc/`) |
| `apx setup` | Wizard interactivo: proveedor → modelo → canales → daemon (alias: `install`) |
| `apx status` | Estado completo del sistema: daemon, super-agent, engines, Telegram, proyectos |
| `apx update` | Verifica actualizaciones y hace upgrade (alias: `upgrade`) |

---

## Proyectos

| Comando | Qué hace |
|---|---|
| `apx project add [path]` | Registra un proyecto en el daemon |
| `apx project list` | Lista los proyectos registrados |
| `apx project remove <id>` | Desregistra un proyecto |
| `apx project rebuild [id]` | Reconstruye el índice del proyecto desde el filesystem |

---

## Agentes

| Comando | Qué hace |
|---|---|
| `apx agent add <slug>` | Crea un agente local en el proyecto (`--role R --model M --skills a,b --language L --description D`) |
| `apx agent list` | Lista los agentes del proyecto |
| `apx agent get <slug>` | Muestra la definición de un agente |
| `apx agent import` | Importa un template de agente desde `~/.apx/agents/` |
| `apx agent vault list` | Lista los templates del vault (defaults bundled + los tuyos) |
| `apx agent vault add` | Crea un nuevo template en el vault |

---

## Perfil APX

| Comando | Qué hace |
|---|---|
| `apx identity show` | Muestra la identidad actual de APX (nombre, dueño, personalidad) |
| `apx identity set <k> <v>` | Establece un campo de la identidad |
| `apx identity wizard` | Wizard interactivo de identidad |

---

## Config

| Comando | Qué hace |
|---|---|
| `apx config show` | Muestra la config del proyecto (`--effective` para la vista merged, `--only-overrides` para los deltas) |
| `apx config set <key.path> <value>` | Establece un valor JSON-aware en `.apc/config.json` |
| `apx config unset <key.path>` | Elimina una clave |
| `apx model status` | Estado de salud de los proveedores + modelo activo (fallback router) |
| `apx model order <p1> <p2> …` | Establece el orden de fallback — ej. `ollama openrouter groq` |
| `apx model key <provider> <api-key>` | Guarda una API key en `~/.apx/config.json` |
| `apx model set <provider> <model-id>` | Fija un model id específico para un proveedor |
| `apx model test` | Resuelve qué modelo elegiría el router ahora mismo |
| `apx model enable` | Habilita el fallback router |
| `apx model disable` | Deshabilita el fallback router (solo primary) |
| `apx permission show` | Muestra el modo de permisos de herramientas de APX |
| `apx permission set <mode>` | Establece el modo: `total` \| `automatico` \| `permiso` |

## Memoria

| Comando | Qué hace |
|---|---|
| `apx memory <slug>` | Muestra el `memory.md` de un agente |
| `apx memory <slug> --append "<note>"` | Agrega una nota bajo `## Recent context` |
| `apx memory <slug> --replace` | Reemplaza la memoria desde stdin |

## Sesiones

| Comando | Qué hace |
|---|---|
| `apx session new <slug>` | Crea un archivo de sesión (`--title T --task-ref REF --body -\|"text"`) |
| `apx session list [slug]` | Lista sesiones (`--last N`) |
| `apx session get <id>` | Muestra metadata o cuerpo de una sesión (`--body`) |
| `apx session update <id>` | Actualiza el frontmatter de una sesión (`--status S --result R --title T`) |
| `apx session close <id>` | Marca una sesión como completa (`--result "text"`) |
| `apx session check` | Sale con error si ya existe una sesión activa |
| `apx session close-stale` | Cierra automáticamente sesiones con más de 1 h de antigüedad |
| `apx session resume <id>` | Muestra la sesión APC + transcript opcional (`--summary --full`) |
| `apx session compact <slug>` | Colapsa el historial de conversación en un resumen |
| `apx session find <text>` | Busca sesiones por título entre todos los engines (`--deep --engine X`) |
| `apx session summary <id>` | Resumen LLM de cualquier sesión por id |
| `apx session ask <id> "<q>"` | Hace una pregunta sobre una sesión (`--max-chunks N`) |
| `apx sessions list` | Lista sesiones de los AI engines (`--engine claude\|codex\|apx --project P`) |

## MCPs

| Comando | Qué hace |
|---|---|
| `apx mcp list` | Lista los servidores MCP del proyecto |
| `apx mcp add <name>` | Agrega un servidor MCP al proyecto (`--command <cmd> --env KEY=VAL`) |
| `apx mcp remove <name>` | Elimina un servidor MCP del proyecto |
| `apx mcp enable <name>` | Habilita un servidor MCP del proyecto |
| `apx mcp disable <name>` | Deshabilita un servidor MCP del proyecto |
| `apx mcp run <name> <tool>` | Llama a una herramienta MCP (`[<json-args>]`) |
| `apx mcp tools <name>` | Muestra la lista de herramientas de un servidor |
| `apx mcp check` | Audita el merge multi-fuente y los conflictos |

## Servicio Daemon

| Comando | Qué hace |
|---|---|
| `apx daemon start` | Inicia el daemon |
| `apx daemon stop` | Detiene el daemon |
| `apx daemon restart` | Detiene y reinicia — levanta cambios de código, prompts y herramientas |
| `apx daemon reload` | Recarga solo `~/.apx/config.json` (sin restart, sin reload de código) |
| `apx daemon status` | Muestra la salud del daemon |
| `apx daemon logs` | Muestra el tail del log del daemon (`--tail N`) |
| `apx log` | Log unificado de todos los módulos (`-f` follow, `--tail N`, `--errors` only) |

<Aside type="tip">
  `apx log -f --errors` transmite solo las líneas `[ERROR]` en tiempo real — la forma más rápida de ver qué está fallando.
</Aside>

## Telegram

| Comando | Qué hace |
|---|---|
| `apx telegram send "text"` | Envía un mensaje (`--chat <id>`) |
| `apx telegram status` | Muestra el estado del plugin de Telegram |
| `apx telegram start` | Inicia el polling en todos los canales configurados |
| `apx telegram stop` | Detiene el polling (la config se preserva) |
| `apx telegram setup` | Muestra las instrucciones de configuración |
| `apx telegram channel <sub>` | Gestiona canales: `add \| list \| show \| set \| unset \| remove` |
| `apx telegram contacts` | Lista el registro de contactos conocidos |
| `apx telegram role <user_id> <role>` | Asigna un rol a un contacto |
| `apx telegram roles <sub>` | Define rol→herramientas: `list \| set <name> --tools a,b \| rm <name>` |
| `apx telegram owner <channel> <user_id>` | Establece el dueño de un canal |

## Mensajes

| Comando | Qué hace |
|---|---|
| `apx messages tail` | Muestra los mensajes recientes (`--agent <slug> --channel <ch> -n 50 --global`) |
| `apx messages chat` | Transcript de chat con tipos de actores (`--channel telegram -n 50`) |
| `apx messages search "q"` | Busca en los logs de mensajes |

---

## LLM / Code

| Comando | Qué hace |
|---|---|
| `apx code` | Asistente de código terminal de APX (TUI) |
| `apx exec "prompt"` | Llamada LLM one-shot al super-agent (`--model <id> -a <agent>`) |
| `apx chat <agent>` | REPL interactivo del agente (`--conversation <id>`) |
| `apx search "query"` | Búsqueda web via DDG / Brave / browser (`--mode <m> -n N`) |
| `apx conversations list` | Lista las conversaciones almacenadas |
| `apx conversations get <agent> <id>` | Muestra una conversación almacenada |

---

## Runtimes

| Comando | Qué hace |
|---|---|
| `apx run <agent>` | Lanza una sesión completa de runtime externo (`--runtime <id> "prompt" --timeout <s>`) |
| `apx env detect` | Muestra qué CLIs de runtime están instalados |

**Runtimes disponibles:** `claude-code` · `codex` · `opencode` · `aider` · `cursor-agent` · `gemini-cli` · `qwen-code`

## Agent-to-Agent

| Comando | Qué hace |
|---|---|
| `apx send <from> <to> "msg"` | Registra un mensaje A2A (`--deliver` ejecuta el engine destino) |
| `apx connections <agent>` | Mapa mental: quién, cómo y cuándo comunicó este agente |

## Routines

| Comando | Qué hace |
|---|---|
| `apx routine list` | Lista las routines con próxima y última ejecución |
| `apx routine add <name>` | Crea una routine |
| `apx routine get <name>` | Muestra la definición de una routine |
| `apx routine history <name>` | Muestra el historial de ejecución |
| `apx routine run <name>` | Dispara manualmente (ignora el schedule) |
| `apx routine enable <name>` | Habilita una routine |
| `apx routine disable <name>` | Deshabilita una routine |
| `apx routine remove <name>` | Elimina una routine |

---

## Tasks

| Comando | Qué hace |
|---|---|
| `apx task add "<title>"` | Crea una tarea (`--tag t --due 2026-06-01 --agent A --body "..."`) |
| `apx task list` | Lista tareas (`--state open\|done\|dropped\|all --tag X --agent Y --limit N`) |
| `apx task show <id>` | Muestra una tarea como JSON |
| `apx task done <id>` | Marca la tarea como hecha (`--by <name>`) |
| `apx task drop <id>` | Descarta / archiva una tarea |
| `apx task reopen <id>` | Reabre una tarea terminada o descartada |
| `apx task patch <id>` | Edita campos (`--title T --body B --due D --agent A --tag t`) |

## Artifacts

| Comando | Qué hace |
|---|---|
| `apx artifact create <name>` | Crea un archivo gestionado en el storage del proyecto (`--content '...' --project 0`) |
| `apx artifact list` | Lista los artifacts |
| `apx artifact show <name>` | Muestra el contenido de un artifact |
| `apx artifact remove <name>` | Elimina un artifact |

---

## Commands & Skills

| Comando | Qué hace |
|---|---|
| `apx command list` | Lista los workflow commands en `.apc/commands/` |
| `apx command show <name>` | Muestra el contenido de un command |
| `apx skills add [targets]` | Instala skills en los rule files del IDE (`claude-code cursor windsurf copilot trae`) |
| `apx skills add --global` | Instala en `~/.claude/skills/` etc. |
| `apx skills list` | Lista skills en `.apc/skills/` del proyecto |
| `apx skills status` | Muestra qué targets del IDE tienen los skills bundled instalados |
| `apx skills sync` | Fuerza el refresh de todos los skills bundled a cada directorio global |

---

## Plugins

| Comando | Qué hace |
|---|---|
| `apx plugins list` | Muestra los plugins cargados y su estado |
| `apx plugins status <id>` | Estado detallado de un plugin (ej. `telegram`) |

## Voice & Desktop

| Comando | Qué hace |
|---|---|
| `apx voice say "text"` | TTS via daemon (`--provider <id> --voice <name> --no-play`) |
| `apx voice listen` | Mic → `/voice/turn` → respuesta (`--seconds N --no-play`) |
| `apx voice providers` | Lista los proveedores TTS / STT configurados |
| `apx desktop start` | Lanza la ventana flotante Electron de voz (`--debug`) |
| `apx desktop stop` | Detiene la ventana desktop |
| `apx desktop status` | Muestra estado del proceso, PID y estado de autostart |
| `apx desktop install` | Activa el autostart al login de usuario (sin sudo) |
| `apx desktop uninstall` | Desactiva el autostart |

---

## Pair (Dispositivos Compañeros)

| Comando | Qué hace |
|---|---|
| `apx pair [label]` | QR pairing para la app Deck / clientes compañeros |
| `apx pair web` | QR + link para emparejar un browser en la LAN |
| `apx pair list` | Lista los clientes emparejados |
| `apx pair revoke <id>` | Revoca el token de un cliente emparejado |

## Flags Globales

| Flag | Efecto |
|---|---|
| `--project <name\|id\|path>` | Fija un comando a un proyecto específico |
| `--help` | Muestra la ayuda de cualquier comando |
| `--version` | Imprime la versión |
