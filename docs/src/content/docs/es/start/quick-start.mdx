Este recorrido te lleva desde un proyecto vacío hasta un agente en ejecución en pocos minutos.

<Steps>

1. **Inicializá un proyecto.** En cualquier directorio que quieras que APX gestione:

   ```bash
   apx init --name "My Project"
   ```

   Esto crea la carpeta `.apc/` y un `AGENTS.md`. Mirá
   [Proyectos](/apx/docs/es/concepts/projects/) para ver qué queda en disco.

2. **Agregá un agente.** Dale un slug, un rol y un modelo:

   ```bash
   apx agent add sofia --role "backend reviewer" --model ollama:llama3.2:3b
   ```

   <Terminal command="apx agent add sofia --role 'backend reviewer' --model ollama:llama3.2:3b" caption="apx agent add creando el agente sofia y escribiendo .apc/agents/sofia.md">
   {`Added agent sofia
     Role: backend reviewer
     Model: ollama:llama3.2:3b
   ✓ wrote .apc/agents/sofia.md
   ✓ created runtime dir .apc/runtime/agents/sofia/
   • rebuilt project #1 (5 agents)`}
   </Terminal>

3. **Ejecutá el agente a través de un runtime completo.** Esto lanza una CLI de programación externa (acá, Claude
   Code) con el system prompt del agente inyectado:

   ```bash
   apx run sofia --runtime claude-code "Review the open PRs and summarize them"
   ```

   ¿Preferís una respuesta rápida de una sola vez del LLM? Usá `exec`:

   ```bash
   apx exec sofia "What is my role in this project?"
   ```

4. **Observá lo que pasa.** Seguí el registro de mensajes a través de todos los canales:

   ```bash
   apx messages tail
   ```

   <Terminal command="apx messages tail" caption="apx messages tail mostrando actividad de runtime y exec con los tipos de actor">
   {`2026-06-14 09:31:02  → runtime    sofia: spawned claude-code (exit 0, 4.2s)
   2026-06-14 09:31:06  ← user       alex: What is my role in this project?
   2026-06-14 09:31:08  → agent      sofia: I'm the backend reviewer — I check PRs
   2026-06-14 09:32:14  → super-agent roby: routed 1 task to sofia
   • 4 mensajes · canal: all`}
   </Terminal>

</Steps>

## El asistente interactivo

¿Preferís que te guíen? `apx setup` te lleva por provider → modelo → canales → daemon en un solo
flujo:

```bash
apx setup
```

<Aside type="note">
	`apx run` usa **runtimes** externos (CLIs de programación completas). `apx exec` usa **engines** (llamadas
	directas al LLM). La diferencia se cubre en [Runtimes](/apx/docs/es/engine/runtimes/) y
	[Engines](/apx/docs/es/engine/engines/).
</Aside>
