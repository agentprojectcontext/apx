APX se distribuye como un único paquete npm que provee tres binarios: `apx` (CLI), `apx-daemon`
(el servidor local) y `apx-mcp` (el puente de servidor MCP).

## Requisitos

- **Node.js 20+** (el campo engines del paquete permite 18+, pero se recomienda 20+).
- macOS, Linux o WSL. Algunas superficies (Desktop, Voz) tienen dependencias extra a nivel del SO,
  cubiertas en sus propias páginas.

## Instalar

<Tabs syncKey="pkg">
	<TabItem label="npm">
		```bash
		npm install -g apx
		```
	</TabItem>
	<TabItem label="pnpm">
		```bash
		pnpm add -g apx
		```
	</TabItem>
</Tabs>

<Aside type="tip">
	El daemon arranca automáticamente en tu primer llamado a `apx` — no hace falta lanzarlo a mano.
</Aside>

## Verificar

<Steps>

1. Verificá que la CLI esté en tu PATH:

   ```bash
   apx --version
   ```

2. Ejecutá cualquier comando para levantar el daemon, después confirmá que está sano:

   ```bash
   apx status
   ```

   <Screenshot
     surface="terminal"
     caption="apx status mostrando el daemon corriendo y su puerto"

</Steps>

## Qué se crea

La primera ejecución crea el directorio de runtime local:

```text
~/.apx/
├── config.json        ← engines, providers, settings globales
├── identity.json      ← el nombre visible del asistente (default "APX")
└── projects/          ← estado de runtime por proyecto (sesiones, mensajes, cachés)
```

Nada de esto se commitea — es local a tu máquina. Ver
[Configuración](/apx/es/concepts/configuration/) para la referencia completa.
