APX se distribuye como un único paquete npm que provee tres binarios: `apx` (CLI), `apx-daemon`
(el servidor local) y `apx-mcp` (el puente del servidor MCP).

## Requisitos

- **Node.js 20+** (el campo engines del paquete permite 18+, pero se recomienda 20+).
- macOS, Linux o WSL. Algunas superficies (Desktop, Voice) tienen dependencias extra a nivel del SO cubiertas en
  sus propias páginas.

## Instalar

<Tabs syncKey="pkg">
	<TabItem label="npm">
		```bash
		npm install -g @agentprojectcontext/apx
		```
	</TabItem>
	<TabItem label="pnpm">
		```bash
		pnpm add -g @agentprojectcontext/apx
		```
	</TabItem>
</Tabs>

<Aside type="tip">
	El daemon arranca automáticamente en tu primera llamada a `apx` — no necesitás lanzarlo a mano.
</Aside>

## Verificar

<Steps>

1. Comprobá que el CLI esté en tu PATH:

   ```bash
   apx --version
   ```

2. Ejecutá cualquier comando para arrancar el daemon, luego confirmá que esté sano:

   ```bash
   apx status
   ```

## Qué se crea

La primera ejecución crea el directorio local del runtime:

```text
~/.apx/
├── config.json        ← engines, providers, global settings
├── identity.json      ← the assistant's user-facing name (default "APX")
└── projects/          ← per-project runtime state (sessions, messages, caches)
```

Nada de esto se commitea — es local a tu máquina. Mirá
[Configuración](/apx/docs/es/concepts/configuration/) para la referencia completa.
