APX tiene una capa de voz unificada: una facade TTS (texto a voz) para hablar
texto y un sidecar STT (voz a texto) basado en Whisper para transcribir el audio
del micrófono. La CLI `apx voice` expone ambos. La [ventana Desktop](/apx/es/surfaces/desktop/)
y el [Deck](/apx/es/surfaces/deck/) usan los mismos motores internamente.

## Comandos

```bash
# Sintetizar y reproducir
apx voice say "Hola, mundo!"
apx voice say "Hola" --provider piper
apx voice say "Hola" --provider gemini --voice Aoede
apx voice say "Hola" --no-play          # genera el archivo, no lo reproduce

# Escuchar (mic → STT → super-agente → respuesta TTS)
apx voice listen                          # para al detectar silencio (requiere sox)
apx voice listen --seconds 5             # captura de duración fija
apx voice listen --seconds 5 --no-play  # transcribe + agente, sin reproducción de audio

# Listar proveedores configurados
apx voice providers
```

`--provider` sobreescribe el defecto configurado solo para esa llamada.
`--voice` sobreescribe el ID de voz dentro de un proveedor (soportado por
OpenAI, ElevenLabs y Gemini). `--no-play` escribe el archivo de audio en
`~/.apx/tmp/tts/` e imprime la ruta sin reproducirlo.

## Proveedores

| ID | ¿Local? | ¿Necesita clave? | Calidad | Notas |
|---|---|---|---|---|
| `piper` | sí | no | Buena | Recomendado por defecto. Requiere binario `piper` + modelo `.onnx`. |
| `elevenlabs` | no | sí | Excelente | Tier free: 10 k chars/mes. Modelo `eleven_multilingual_v2`. |
| `openai` | no | sí | Buena | Reutiliza `engines.openai.api_key`. Modelo `tts-1`. |
| `gemini` | no | sí | Buena | Devuelve L16 PCM raw — APX lo envuelve en WAV automáticamente. |
| `mock` | sí | no | Silencio | WAV silencioso. Solo útil para tests. |

El proveedor `auto` prueba en orden: `piper → elevenlabs → openai → gemini → mock`.
En una instalación nueva sin proveedor configurado, `auto` cae en `mock`
(silencio). Configurá al menos un proveedor real antes de esperar audio.

<Aside type="tip">
  **Piper es el defecto recomendado** para voz local, offline y sin clave.
  Ver decisión `spec/decisions/004-piper-default-local-tts.md`. La voz
  argentina `es_AR-daniela-high` tiene la mayor calidad publicada para es-AR.
</Aside>

## Configuración

La configuración de proveedores vive en `~/.apx/config.json` bajo `voice.tts`:

```json
{
  "voice": {
    "tts": {
      "provider": "piper",
      "piper": {
        "bin": "piper",
        "model": "/home/vos/.apx/voices/es_AR-daniela-high.onnx"
      },
      "elevenlabs": {
        "api_key": "...",
        "model": "eleven_multilingual_v2",
        "voice_id": "..."
      },
      "openai": {
        "api_key": "...",
        "model": "tts-1",
        "voice": "alloy",
        "format": "mp3"
      },
      "gemini": {
        "api_key": "...",
        "model": "gemini-2.5-flash-preview-tts",
        "voice": "Aoede"
      }
    }
  }
}
```

Cambiá el proveedor activo con:

```bash
apx config set voice.tts.provider piper
```

También podés gestionar los proveedores desde el [panel web](/apx/es/surfaces/web/)
en **Voces** (`/m/voice`).

## Guías de configuración

### Piper (local, sin internet)

Piper corre completamente offline. Necesitás el binario `piper` y un modelo de
voz `.onnx`.

<Steps>

1. Instalá el binario de Piper:

   <Tabs syncKey="os">
     <TabItem label="macOS (arm64)">
       ```bash
       curl -L https://github.com/rhasspy/piper/releases/latest/download/piper_macos_aarch64.tar.gz \
         -o /tmp/piper.tar.gz
       sudo tar xzf /tmp/piper.tar.gz -C /usr/local/bin --strip-components=1
       ```
     </TabItem>
     <TabItem label="Linux (x86_64)">
       ```bash
       curl -L https://github.com/rhasspy/piper/releases/latest/download/piper_linux_x86_64.tar.gz \
         -o /tmp/piper.tar.gz
       sudo tar xzf /tmp/piper.tar.gz -C /usr/local/bin --strip-components=1
       ```
     </TabItem>
   </Tabs>

2. Descargá un modelo de voz. Para español argentino (recomendado):

   ```bash
   mkdir -p ~/.apx/voices
   cd ~/.apx/voices
   curl -LO https://huggingface.co/rhasspy/piper-voices/resolve/main/es/es_AR/daniela/high/es_AR-daniela-high.onnx
   curl -LO https://huggingface.co/rhasspy/piper-voices/resolve/main/es/es_AR/daniela/high/es_AR-daniela-high.onnx.json
   ```

3. Apuntale a APX:

   ```bash
   apx config set voice.tts.provider piper
   apx config set voice.tts.piper.model "$HOME/.apx/voices/es_AR-daniela-high.onnx"
   ```

4. Probá:

   ```bash
   apx voice say "hola mundo" --provider piper
   ```

</Steps>

### Gemini cloud (más rápido si ya tenés una clave)

```bash
apx config set voice.tts.provider gemini
apx config set voice.tts.gemini.api_key '<TU_CLAVE_GEMINI>'
apx voice say "hola mundo" --provider gemini
```

<Aside type="note">
  Gemini TTS devuelve L16 PCM raw, no MP3. APX lo envuelve automáticamente en
  un header WAV. Los archivos de salida son `.wav`; si necesitás MP3 usá
  `ffmpeg` para convertir.
</Aside>

### ElevenLabs

```bash
apx config set voice.tts.provider elevenlabs
apx config set voice.tts.elevenlabs.api_key '<TU_CLAVE_11L>'
apx config set voice.tts.elevenlabs.voice_id '<ID_DE_VOZ>'
apx voice say "hola mundo" --provider elevenlabs
```

## El endpoint unificado de turno de voz

`POST /voice/turn` es un único round-trip bidireccional:

1. STT — transcribe el audio entrante (o acepta `text` directamente, saltando el STT).
2. Agente — ejecuta el super-agente sobre el texto transcripto.
3. TTS — sintetiza la respuesta y devuelve la ruta al archivo de audio.

```bash
# Llamar con curl con texto ya transcripto
curl -X POST http://127.0.0.1:7430/voice/turn \
  -H "Authorization: Bearer $(cat ~/.apx/daemon.token)" \
  -H "Content-Type: application/json" \
  -d '{"text": "¿Qué tareas hay abiertas?", "channel": "voice"}'
```

Respuesta:

```json
{
  "user_text": "¿Qué tareas hay abiertas?",
  "reply_text": "Tenés 3 tareas abiertas…",
  "reply_audio_path": "/home/vos/.apx/tmp/tts/reply-abc123.wav",
  "reply_duration_s": 4.1,
  "reply_mime": "audio/wav",
  "provider": "piper"
}
```

La [ventana Desktop](/apx/es/surfaces/desktop/) y el Deck usan este endpoint
internamente. El campo `channel` (`voice`, `deck`, `desktop`, `telegram`)
controla el formato de respuesta: los canales voice/deck reciben respuestas
cortas optimizadas para ser habladas; telegram recibe texto en Markdown.

## Reproducción

`apx voice say` usa el primer reproductor de sistema disponible:
`afplay` (macOS) → `paplay` → `aplay` → `play` (sox) → `ffplay`.

Si no se encuentra ninguno, el archivo se escribe y se imprime la ruta — no
hay error, pero tampoco sonido. Instalá cualquiera de los anteriores para
tener reproducción.

## Troubleshooting

**`apx voice providers` muestra solo `mock`**

No hay ningún proveedor real configurado. Seguí una de las guías de
configuración de arriba.

**El archivo se genera pero no hay sonido**

No hay ningún reproductor de audio del sistema en el PATH. En macOS `afplay`
siempre está disponible. En Linux instalá `sox` (`apt install sox`) o
`pulseaudio-utils`.

**El output de Gemini suena distorsionado o no reproduce**

Versiones antiguas de APX no envolvían el L16 PCM raw en un header WAV.
Actualizá APX o convertí con `ffmpeg -f s16le -ar 24000 -ac 1 -i raw.pcm out.wav`.

**`apx voice listen` devuelve texto vacío**

`sox` es requerido para grabar con detección de silencio. Instalalo o usá
`--seconds N` para especificar una duración fija en su lugar.
