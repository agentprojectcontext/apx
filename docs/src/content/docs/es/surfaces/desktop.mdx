La superficie **Desktop** es una pequeña app Electron que flota sobre todas tus
ventanas. Te da una cápsula de voz persistente y accesible con el teclado:
presioná el atajo global y empezá a hablar. La respuesta se muestra como texto
en streaming y, si tenés un proveedor TTS configurado, también se reproduce como
audio.

## Comandos

```bash
apx desktop start           # lanzar la ventana flotante
apx desktop start --debug   # lanzar con logs de arranque detallados
apx desktop stop            # cerrar la ventana
apx desktop status          # mostrar estado del proceso, PID, flag de autostart
apx desktop install         # registrar autostart al inicio de sesión (sin sudo)
apx desktop uninstall       # eliminar la entrada de autostart
```

`start` es idempotente: si la ventana ya está corriendo, reporta el estado
actual en lugar de abrir una segunda copia.

## Autostart al iniciar sesión

`apx desktop install` registra un login item por usuario para que la ventana
se abra automáticamente cuando iniciás sesión. No se necesita contraseña de
administrador ni sudo.

```bash
apx desktop install
# Autostart registrado.

apx desktop status
# state: running | pid: 12345 | autostart: enabled
```

Para eliminarlo:

```bash
apx desktop uninstall
```

El mecanismo según plataforma:

| Plataforma | Mecanismo |
|---|---|
| macOS | `~/Library/LaunchAgents/<plist>` |
| Linux | `~/.config/autostart/<desktop>` |
| Windows | Clave de registro run (HKCU) |

## Qué hace la ventana

La ventana es una **cápsula** — una barra flotante estrecha con un único
objetivo: enviar un mensaje al super-agente y reproducir la respuesta.

### Máquina de estados

La cápsula pasa por cinco estados:

| Estado | Qué ves |
|---|---|
| `idle` | Campo de texto + botón de micrófono. Enter para enviar texto, click en mic (o mantener el atajo) para grabar. |
| `listening` | Waveform de audio en vivo driven por la amplitud real del micrófono. Cancelar o tocar Enviar para confirmar. |
| `transcribing` | Dots animados — el audio WebM/OGG está siendo decodificado por el sidecar STT de Whisper. |
| `thinking` | Dots + shimmer — el super-agente está generando tokens, se muestran a medida que llegan. |
| `speaking` | Avatar del agente + shimmer — se está reproduciendo el audio TTS. Aparece un scrubber en la transcripción. |

### Transcripción de la conversación

Cuando un turno termina, la card de transcripción se expande debajo de la
cápsula. Cada turno del agente muestra el texto con animación word-by-word y,
si TTS está configurado, un scrubber de waveform con play/pause y click para
saltar. Las llamadas a herramientas aparecen como pills con spinner mientras
corren y checkmark al completar.

### Atajo global

El atajo por defecto es `Cmd+G` en macOS y `Ctrl+G` en Windows/Linux.
Podés sobreescribirlo en `~/.apx/config.json`:

```json
{
  "desktop": {
    "shortcut": "CommandOrControl+Shift+Space"
  }
}
```

También podés editar el atajo desde el [panel web](/apx/es/surfaces/web/)
en **Escritorio** (`/m/desktop`) sin tocar JSON. Reiniciá la ventana para que
el nuevo atajo tome efecto.

### Posición y tema

```json
{
  "desktop": {
    "position": "right",
    "theme": "light"
  }
}
```

`position` puede ser `"left"`, `"center"` o `"right"` (defecto `"right"`).
`theme` puede ser `"light"` o `"dark"` (defecto `"light"`).

## Cómo la ventana habla con el daemon

El proceso principal de Electron abre un WebSocket a `ws://127.0.0.1:7430` y
envía texto via `POST /desktop/message`. El plugin de desktop del daemon enruta
el mensaje al super-agente y le hace streaming de los tokens de vuelta por
WebSocket:

```text
Desktop (Electron)
  │
  ├── POST /desktop/message { text }  →  el daemon responde inmediatamente
  │
  └── WebSocket ← el daemon hace streaming:
        { type: "thinking" }
        { type: "token", text: "…" }
        { type: "tool_start", name: "create_task" }
        { type: "tool_done",  name: "create_task" }
        { type: "done", text: "respuesta completa" }
        { type: "tts-ready", url: "/voice/tts?path=…", duration: 3.2 }
```

El TTS lo sintetiza el daemon después de que el agente termina, luego la URL se
sirve via `GET /voice/tts?path=…`. La ventana la descarga y la reproduce con
la Web Audio API.

<Aside type="tip">
  La calidad del TTS depende del proveedor configurado. Mirá
  [Voice](/apx/es/surfaces/voice/) para configurar Piper (local, sin clave) o
  un motor en la nube.
</Aside>

## Badge de conexión con el daemon

Un pequeño punto rojo aparece en la esquina superior derecha de la ventana
cuando el WebSocket al daemon está caído. Desaparece ni bien se restaura la
conexión (la ventana se reconecta automáticamente).

## Troubleshooting

**La ventana no se abre después de `apx desktop start`**
- Verificá que Electron esté instalado: `npx electron --version`
- Probá `apx desktop start --debug` para logs detallados
- Confirmá que el daemon está accesible: `apx status`

**No hay reproducción de audio**
- La ventana requiere un proveedor TTS. Ejecutá `apx voice providers` para ver
  qué está configurado, luego seguí la [guía de configuración de Voice](/apx/es/surfaces/voice/).

**El atajo no funciona**
- Otra app puede estar usando ese combo de teclas. Cambialo en la configuración
  y reiniciá.
- En macOS, revisá Ajustes del sistema → Privacidad y seguridad → Accesibilidad
  para Electron.
