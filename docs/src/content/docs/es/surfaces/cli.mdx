El binario `apx` es el punto de entrada principal para todo lo que hace APX. Se comunica con el
[daemon](/apx/es/surfaces/daemon/) local por HTTP (`127.0.0.1:7430`) y lo inicia automáticamente
en el primer uso, así que nunca tenés que arrancar el daemon por separado.

## Flags globales

| Flag | Descripción |
|------|-------------|
| `--project <name\|id\|path>` | Fijá todos los comandos a un proyecto específico. Útil cuando trabajás con varios proyectos registrados desde la misma shell. |
| `--help` | Muestra la ayuda del comando o subcomando actual. |
| `--version` | Muestra la versión de APX instalada. |

<Aside type="tip">
  Cada comando que acepta `--project` también acepta el nombre del proyecto, su id numérico o una
  ruta del sistema de archivos. Ejecutá `apx project list` para ver los ids registrados.
</Aside>

## Bootstrap

| Comando | Descripción |
|---------|-------------|
| `apx init [path] [--name "<name>"]` | Inicializa los archivos APC (`.apc/`, `AGENTS.md`) en el directorio actual o en una ruta destino. |
| `apx setup` | Wizard interactivo: provider → modelo → canales → daemon. Alias: `apx install`. |
| `apx status` | Estado completo del sistema: daemon, super-agent, engines, Telegram, proyectos registrados. |
| `apx update` | Verifica si hay una versión publicada de APX y actualiza la instalación global. Alias: `apx upgrade`. |

## Proyectos

| Comando | Descripción |
|---------|-------------|
| `apx project add [path]` | Registra un directorio en el daemon para que rastree sus agentes, sesiones y mensajes. |
| `apx project list` | Lista todos los proyectos registrados en el daemon. |
| `apx project remove <id>` | Elimina un proyecto del daemon (no borra los archivos). |
| `apx project rebuild [id]` | Reconstruye el índice del daemon desde el contexto del proyecto en disco. |
| `apx project config show <project>` | Muestra la config efectiva + la del proyecto. |
| `apx project config set <project> <key> <value>` | Establece una clave en `.apc/config.json`. |
| `apx project config unset <project> <key>` | Elimina una clave de `.apc/config.json`. |
| `apx project config edit <project>` | Abre la config del proyecto en `$EDITOR` para un reemplazo completo al guardar. |

Ver [Proyectos](/apx/es/concepts/projects/).

## Agentes

| Comando | Descripción |
|---------|-------------|
| `apx agent add <slug>` | Crea un agente local del proyecto. Flags: `--role R`, `--model M`, `--skills a,b`, `--language es-AR`, `--description D`. |
| `apx agent list` | Lista los agentes del proyecto actual. |
| `apx agent get <slug>` | Muestra la definición de un agente. |
| `apx agent import <slug>` | Importa una plantilla de agente del vault al proyecto actual. |
| `apx agent vault list` | Lista las plantillas globales del vault (defaults incluidos + tus personalizaciones). |
| `apx agent vault add <slug>` | Crea o promueve un agente al vault bajo `~/.apx/agents/`. |
| `apx agent vault rm <slug>` | Marca una plantilla del vault como eliminada (los defaults incluidos se ocultan, no se borran). |
| `apx agent vault restore <slug>` | Levanta la marca para que un default eliminado vuelva a aparecer. |

Ver [Agentes](/apx/es/concepts/agents/).

## Config

| Comando | Descripción |
|---------|-------------|
| `apx config show [--effective] [--only-overrides]` | Muestra `.apc/config.json` (o la config efectiva combinada). |
| `apx config set <key.path> <value>` | Set con soporte JSON, p. ej. `apx config set engines.openai.model gpt-5.2`. |
| `apx config unset <key.path>` | Elimina una clave de `.apc/config.json`. |
| `apx model status` | Verificación de salud de los providers + modelo activo del router de fallback. |
| `apx model order <p1> <p2> …` | Define el orden de intento de fallback, p. ej. `ollama openrouter groq`. |
| `apx model key <provider> <api-key>` | Guarda una API key para `groq` u `openrouter` en `~/.apx/config.json`. |
| `apx model test` | Resuelve qué modelo elegiría el router en este momento. |
| `apx permission show` | Muestra el modo de permisos de herramientas APX actual. |
| `apx permission set <mode>` | Establece el modo de permisos: `total`, `automatico` o `permiso`. |

Ver [Configuración](/apx/es/concepts/configuration/).

## Sesiones

Las sesiones APX son archivos Markdown locales bajo `~/.apx/projects/<id>/agents/<slug>/sessions/`.

| Comando | Descripción |
|---------|-------------|
| `apx session new <slug> --title "T"` | Crea un archivo de sesión. Opcional: `--task-ref REF`, `--body -\|"text"`. |
| `apx session list [slug] [--last N]` | Lista las sesiones, opcionalmente filtradas por agente. |
| `apx session get <id> [--body]` | Muestra los metadatos de la sesión o el cuerpo completo. |
| `apx session update <id>` | Actualiza el frontmatter: `--status S`, `--result R`, `--title T`. |
| `apx session close <id> [--result "text"]` | Marca una sesión como completada. |
| `apx session check` | Sale con código 1 si hay una sesión activa (guardia anti-colisión para scripts). |
| `apx session close-stale` | Cierra automáticamente las sesiones con más de 1 hora de antigüedad. |
| `apx session resume <id>` | Muestra una sesión y opcionalmente la continúa: `--summary`, `--full`, `--continue`, `--into apx[:slug]`. |
| `apx session compact <slug>` | Colapsa el historial de conversación en un resumen durable. |
| `apx session find "<text>"` | Busca sesiones por título en todos los engines. `--deep` también busca en el contenido. |
| `apx session summary <id>` | Resumen LLM de cualquier sesión por id (resuelve el engine automáticamente). |
| `apx session ask <id> "<q>"` | Hace una pregunta sobre el transcript de una sesión. `--max-chunks N`. |
| `apx sessions list` | Lista de sesiones multi-engine. `--engine claude\|codex\|apx`, `--project P`, `--dir D`. |

Ver [Sesiones](/apx/es/concepts/sessions/).

## Daemon

| Comando | Descripción |
|---------|-------------|
| `apx daemon start` | Inicia el daemon local. |
| `apx daemon reload` | Vuelve a leer `~/.apx/config.json` sin reiniciar (no recarga código). |
| `apx daemon stop` | Detiene el daemon. |
| `apx daemon status` | Muestra el estado del daemon. |
| `apx daemon logs [--tail N]` | Muestra el tail del log stdout del daemon. |
| `apx log [-f] [--tail N] [--errors]` | Log unificado de todos los módulos (recomendado sobre `daemon logs`). |

## Telegram

| Comando | Descripción |
|---------|-------------|
| `apx telegram send "text" [--chat <id>]` | Envía un mensaje a través del bridge configurado. |
| `apx telegram status` | Muestra el estado del plugin Telegram. |
| `apx telegram setup` | Muestra la guía de configuración. |
| `apx telegram start` | Inicia el polling en todos los canales configurados. |
| `apx telegram stop` | Detiene el polling (la config permanece intacta). |
| `apx telegram channel add` | Wizard interactivo (o forma no interactiva) para agregar un canal. |
| `apx telegram channel list` | Lista los canales configurados. |
| `apx telegram channel show <name>` | Muestra un canal como JSON. |
| `apx telegram channel set <name>` | Actualiza campos del canal: `--project`, `--agent`, `--respond-engine`, `--bot-token`, `--chat-id`. |
| `apx telegram channel unset <name>` | Borra campos opcionales del canal. |
| `apx telegram channel remove <name>` | Elimina un canal. |
| `apx telegram contacts` | Lista el roster de personas conocidas con roles y última actividad. |
| `apx telegram role <user_id> <role>` | Asigna un rol a un contacto por su `user_id` de Telegram. |
| `apx telegram roles` | Lista o gestiona los mapas de capacidades role→tools. |
| `apx telegram owner <channel> <user_id>` | Establece quién es el dueño de un canal. |

Ver [Telegram](/apx/es/capabilities/telegram/).

## LLM / Code

| Comando | Descripción |
|---------|-------------|
| `apx code` | Lanza el asistente de coding terminal APX (TUI). `--project`, `--agent <slug>`. |
| `apx exec "prompt"` | Llamada LLM de un solo shot al super-agent. `-a <agent>` para un slug APC, `--model <id>`. |
| `apx chat <agent>` | REPL interactivo de agente. `--conversation <id>` para continuar un hilo existente. |
| `apx search "query"` | Búsqueda web (DuckDuckGo / Brave / browser). `--mode <m>`, `-n N`. |
| `apx conversations list <agent>` | Lista las conversaciones exec/chat guardadas de un agente. |
| `apx conversations get <agent> <id>` | Muestra una conversación. |

Ver [TUI](/apx/es/surfaces/tui/).

## Routines

| Comando | Descripción |
|---------|-------------|
| `apx routine list` | Lista las rutinas con los tiempos de próxima/última ejecución. |
| `apx routine add <name>` | Crea una rutina. `--kind K`, `--schedule S`, `--spec '{...}'`. |
| `apx routine get <name>` | Muestra la definición de la rutina. |
| `apx routine history <name>` | Muestra el historial de ejecuciones. |
| `apx routine run <name>` | Dispara manualmente (ignora el schedule). |
| `apx routine enable <name>` | Habilita una rutina desactivada. |
| `apx routine disable <name>` | Desactiva una rutina sin eliminarla. |
| `apx routine remove <name>` | Elimina una rutina. |

**Kinds:** `heartbeat`, `exec_agent`, `super_agent`, `telegram`, `shell`.

**Schedules:** `every:60s`, `every:5m`, `every:1h`, `once:<iso-8601>`.

Ver [Routines](/apx/es/capabilities/routines/).
