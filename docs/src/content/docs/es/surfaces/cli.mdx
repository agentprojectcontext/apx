El binario `apx` es el punto de entrada principal para todo lo que hace APX. Se comunica con el
[daemon](/apx/docs/es/surfaces/daemon/) local sobre HTTP (`127.0.0.1:7430`) y lo arranca automáticamente en el primer uso, así que
nunca necesitás iniciar el daemon por separado.

## Flags globales

| Flag | Descripción |
|------|-------------|
| `--project <name\|id\|path>` | Fija cada comando a un proyecto específico. Útil cuando trabajás con varios proyectos registrados desde una sola terminal. |
| `--help` | Imprime la ayuda del comando o subcomando actual. |
| `--version` | Imprime la versión de APX instalada. |

<Aside type="tip">
  Todo comando que acepta `--project` también acepta el nombre visible del proyecto, su id numérico, o
  una ruta de archivos. Ejecutá `apx project list` para ver los ids registrados.
</Aside>

## Bootstrap

| Comando | Descripción |
|---------|-------------|
| `apx init [path] [--name "<name>"]` | Inicializa los archivos de proyecto APC (`.apc/`, `AGENTS.md`) en el directorio actual o en una ruta de destino. |
| `apx setup` | Asistente interactivo: proveedor → modelo → canales → daemon. Alias: `apx install`. |
| `apx status` | Estado completo del sistema: daemon, super-agente, motores, Telegram, proyectos registrados. |
| `apx update` | Verifica si hay una actualización publicada de APX y actualiza la instalación global. Alias: `apx upgrade`. |

## Proyectos

| Comando | Descripción |
|---------|-------------|
| `apx project add [path]` | Registra un directorio en el daemon para que rastree sus agentes, sesiones y mensajes. |
| `apx project list` | Lista todos los proyectos registrados en el daemon. |
| `apx project remove <id>` | Elimina una entrada de proyecto del almacenamiento del daemon (no borra archivos). |
| `apx project rebuild [id]` | Reconstruye el índice del daemon a partir del contexto del sistema de archivos del proyecto. |
| `apx project config show <project>` | Imprime la configuración efectiva + la específica del proyecto. |
| `apx project config set <project> <key> <value>` | Establece una clave con notación de puntos en `.apc/config.json`. |
| `apx project config unset <project> <key>` | Elimina una clave con notación de puntos de `.apc/config.json`. |
| `apx project config edit <project>` | Abre la config del proyecto en `$EDITOR` para un reemplazo completo al guardar. |

Mirá [Proyectos](/apx/docs/es/concepts/projects/) para conocer el layout completo de `.apc/`.

## Agentes

| Comando | Descripción |
|---------|-------------|
| `apx agent add <slug>` | Crea un agente local del proyecto. Flags: `--role R`, `--model M`, `--skills a,b`, `--language es-AR`, `--description D`. |
| `apx agent list` | Lista los agentes del proyecto actual. |
| `apx agent get <slug>` | Imprime la definición de un agente. |
| `apx agent import <slug>` | Importa una plantilla de agente desde el vault al proyecto actual. |
| `apx agent vault list` | Lista las plantillas del vault global (defaults incluidos + overrides del usuario). |
| `apx agent vault add <slug>` | Crea o promueve un agente al vault bajo `~/.apx/agents/`. |
| `apx agent vault rm <slug>` | Marca una plantilla del vault como eliminada (los defaults incluidos se ocultan, no se borran). |
| `apx agent vault restore <slug>` | Levanta la marca de eliminación para que un default incluido previamente removido vuelva a aparecer. |

Mirá [Agentes](/apx/docs/es/concepts/agents/).

## Config

| Comando | Descripción |
|---------|-------------|
| `apx config show [--effective] [--only-overrides]` | Imprime `.apc/config.json` (o la config efectiva combinada). |
| `apx config set <key.path> <value>` | Set con reconocimiento de JSON, por ejemplo `apx config set engines.openai.model gpt-5.2`. |
| `apx config unset <key.path>` | Elimina una clave de `.apc/config.json`. |
| `apx model status` | Chequeo de salud del proveedor + modelo activo desde el router de fallback. |
| `apx model order <p1> <p2> …` | Establece el orden de intentos del fallback, por ejemplo `ollama openrouter groq`. |
| `apx model key <provider> <api-key>` | Almacena una API key para `groq` o `openrouter` en `~/.apx/config.json`. |
| `apx model test` | Resuelve qué modelo elige el router en este momento. |
| `apx permission show` | Muestra el modo de permisos de herramientas actual de APX. |
| `apx permission set <mode>` | Establece el modo de permisos: `total`, `automatico` o `permiso`. |

Mirá [Configuración](/apx/docs/es/concepts/configuration/).

## Sesiones

Las sesiones de APX son archivos Markdown locales bajo `~/.apx/projects/<id>/agents/<slug>/sessions/`.

| Comando | Descripción |
|---------|-------------|
| `apx session new <slug> --title "T"` | Crea un archivo de sesión. Opcional: `--task-ref REF`, `--body -\|"text"`. |
| `apx session list [slug] [--last N]` | Lista sesiones, opcionalmente acotadas a un agente. |
| `apx session get <id> [--body]` | Imprime los metadatos de la sesión o el cuerpo completo. |
| `apx session update <id>` | Actualiza el frontmatter: `--status S`, `--result R`, `--title T`. |
| `apx session close <id> [--result "text"]` | Marca una sesión como completada. |
| `apx session check` | Sale con 1 si existe una sesión activa (guard de colisión para scripts). |
| `apx session close-stale` | Cierra automáticamente las sesiones de más de 1 hora. |
| `apx session resume <id>` | Muestra una sesión y opcionalmente la continúa: `--summary`, `--full`, `--continue`, `--into apx[:slug]`. |
| `apx session compact <slug>` | Colapsa el historial de la conversación en un resumen duradero. |
| `apx session find "<text>"` | Encuentra sesiones por título en todos los motores. `--deep` también busca en el contenido. |
| `apx session summary <id>` | Resumen por LLM de cualquier sesión por id (resuelve el motor automáticamente). |
| `apx session ask <id> "<q>"` | Hace una pregunta libre sobre la transcripción de una sesión. `--max-chunks N`. |
| `apx sessions list` | Lista de sesiones entre motores. `--engine claude\|codex\|apx`, `--project P`, `--dir D`. |

Mirá [Sesiones](/apx/docs/es/concepts/sessions/).

## Daemon

|---------|-------------|
| `apx daemon start` | Inicia el daemon local. |

## Telegram

| Comando | Descripción |
|---------|-------------|
| `apx telegram send "text" [--chat <id>]` | Envía un mensaje a través del puente configurado. |
| `apx telegram status` | Muestra el estado del plugin de Telegram. |
| `apx telegram setup` | Imprime una guía de configuración. |
| `apx telegram start` | Inicia el polling en cada canal configurado. |
| `apx telegram stop` | Detiene el polling (la config queda intacta). |
| `apx telegram channel add` | Asistente interactivo (o no interactivo de un solo paso) para agregar un canal. |
| `apx telegram channel list` | Lista los canales configurados. |
| `apx telegram channel show <name>` | Imprime un canal como JSON. |
| `apx telegram channel set <name>` | Modifica campos del canal: `--project`, `--agent`, `--respond-engine`, `--bot-token`, `--chat-id`. |
| `apx telegram channel unset <name>` | Limpia campos opcionales del canal. |
| `apx telegram channel remove <name>` | Elimina un canal. |
| `apx telegram contacts` | Lista el roster de personas conocidas con roles y última vez vistas. |
| `apx telegram role <user_id> <role>` | Asigna un rol a un contacto por su `user_id` de Telegram. |
| `apx telegram roles` | Lista o administra los mapas de capacidades rol→herramientas. |
| `apx telegram owner <channel> <user_id>` | Establece quién es dueño de un canal. |

Mirá [Telegram](/apx/docs/es/capabilities/telegram/).

## LLM / Código

| Comando | Descripción |
|---------|-------------|
| `apx code` | Lanza el asistente de programación de APX en la terminal (TUI). `--project`, `--agent <slug>`. |
| `apx exec "prompt"` | Llamada LLM de un solo paso al super-agente. `-a <agent>` para un slug APC, `--model <id>`. |
| `apx chat <agent>` | REPL interactivo del agente. `--conversation <id>` para continuar un hilo existente. |
| `apx search "query"` | Búsqueda web (DuckDuckGo / Brave / navegador). `--mode <m>`, `-n N`. |
| `apx conversations list <agent>` | Lista las conversaciones exec/chat almacenadas de un agente. |
| `apx conversations get <agent> <id>` | Imprime una conversación. |

Mirá [TUI](/apx/docs/es/surfaces/tui/).

## Rutinas

| Comando | Descripción |
|---------|-------------|
| `apx routine list` | Lista las rutinas con sus próximos/últimos tiempos de ejecución. |
| `apx routine add <name>` | Crea una rutina. `--kind K`, `--schedule S`, `--spec '{...}'`. |
| `apx routine get <name>` | Imprime la definición de la rutina. |
| `apx routine history <name>` | Muestra el historial de ejecución. |
| `apx routine run <name>` | La dispara manualmente (ignora el schedule). |
| `apx routine enable <name>` | Habilita una rutina deshabilitada. |
| `apx routine disable <name>` | Deshabilita una rutina sin eliminarla. |
| `apx routine remove <name>` | Elimina una rutina. |

**Schedules:** `every:60s`, `every:5m`, `every:1h`, `once:<iso-8601>`.
