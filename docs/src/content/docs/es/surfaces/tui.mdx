APX trae dos modos interactivos de terminal que van más allá de las llamadas one-shot de `apx exec`:

| Modo | Comando | Ideal para |
|------|---------|------------|
| Asistente de codificación en terminal | `apx code` | Sesiones de codificación multi-turno con contexto completo del sistema, uso de herramientas, cambio de modelo |
| REPL de agentes | `apx chat <agent>` | Conversación de ida y vuelta con un agente específico del proyecto |

Ambos modos persisten los hilos de conversación — podés retomar donde lo dejaste.

---

## `apx code` — asistente de codificación en terminal

`apx code` lanza la TUI de APX: una aplicación de terminal a pantalla completa construida sobre OpenTUI + Solid.js
que transmite respuestas, muestra trazas de herramientas y te deja cambiar de modelo o de agente sin salir
de la sesión.

```bash
apx code                          # default super-agent mode
apx code --project my-project     # pin to a specific project
apx code --agent reviewer         # route chat to a project agent
```

### Qué ofrece la TUI

- **Respuestas en streaming** — el texto fluye progresivamente a medida que el modelo lo genera.
- **Trazas de herramientas** — cada llamada a una herramienta y su resultado aparecen inline en la transcripción.
- **Paleta de comandos** (`Ctrl+P`) — cambiá de modelo o de agente sin salir.
- **Encolar mientras piensa** — escribí tu próximo mensaje mientras el agente sigue respondiendo; se encola
  y se dispara apenas termina la respuesta actual.
- **Interrumpir** (`Ctrl+I` o `Ctrl+C` mientras se ejecuta) — abortá la petición actual; un segundo
  `Ctrl+C` sale.
- **Acciones de mensaje** — hacé clic en un mensaje de usuario para copiarlo, volver a preguntar sobre él, o
  promoverlo al frente de la cola.
- **Scroll** — `↑`/`↓`, `PgUp`/`PgDn`, o `Meta+↑`/`Meta+↓` (saltar al inicio/final).

### Atajos de teclado

| Tecla | Acción |
|-----|--------|
| `Enter` | Enviar mensaje (o encolar si el agente está pensando) |
| `Ctrl+P` | Abrir la paleta de comandos |
| `Ctrl+I` | Interrumpir la petición actual |
| `Ctrl+C` | Interrumpir (primera vez) / Salir (segunda vez) |
| `Ctrl+A` / `Ctrl+E` | Mover el cursor al inicio / final de la entrada |
| `Ctrl+U` | Borrar desde el cursor hasta el inicio de la línea |
| `Ctrl+W` | Borrar la palabra anterior |
| `PgUp` / `PgDn` | Scrollear la transcripción |
| `Meta+↑` / `Meta+↓` | Saltar al inicio / final de la transcripción |
| `Tab` | Ciclar el modo de entrada |
| `exit` o `quit` | Escribir para cerrar |

### Requisitos

`apx code` usa [Bun](https://bun.sh/) para ejecutar directamente el código fuente TypeScript de la TUI. Si `bun` no
está en tu PATH, configurá `BUN_PATH` en tu entorno. Si Bun no está disponible, `apx code` cae
de vuelta al renderizador de terminal incorporado (sin Solid.js).

<Aside type="note">
  Cuando se pasa `--agent <slug>`, la TUI enruta el chat al endpoint `/agents/:slug/chat` del agente
  del proyecto en lugar del stream del super-agente. Las respuestas en modo agente son sin streaming (se muestra
  un spinner mientras carga la respuesta).
</Aside>

## `apx chat` — REPL interactivo de agentes

`apx chat` es un REPL liviano de readline, sin dependencias, para conversaciones multi-turno con un
agente del proyecto. Guarda cada intercambio como un hilo de conversación y te deja retomar uno con
`--conversation <id>`.

```bash
apx chat reviewer                          # new conversation
apx chat reviewer --conversation abc123    # continue an existing thread
apx chat reviewer --model gpt-5.2         # override model for this session
```

{`apx chat · reviewer  (backend reviewer · ollama:llama3.2:3b)
escribí un mensaje · /help para comandos · ⌃D para salir

           correctitud, manejo de errores y cobertura de tests

</Terminal>

El prompt muestra el slug del agente seguido de `>`. Escribí tu mensaje y presioná `Enter`. Salí con
`Ctrl+D`, `exit`, o `quit`.

### Persistencia de la conversación

Cada intercambio se agrega a un archivo de conversación Markdown en:

```text
~/.apx/projects/<project-id>/agents/<slug>/conversations/
```

Cada conversación tiene un id único. Usá `apx conversations list` para verlas y
`apx conversations get` para leer el intercambio completo.

## `apx conversations`

| Comando | Descripción |
|---------|-------------|
| `apx conversations list <agent>` | Lista las conversaciones exec/chat guardadas de un agente. |
| `apx conversations get <agent> <id>` | Imprime una conversación con todos sus turnos. |

```bash
apx conversations list reviewer
apx conversations get reviewer abc123
```

Alias: `apx conv list reviewer`, `apx conv show reviewer abc123`.

## CLI vs TUI

Usá la **CLI** (`apx exec`) para:
- Llamadas one-shot scripteadas desde pipelines de shell o CI.
- Pasar la salida a otras herramientas.
- Registrar un único intercambio en el almacén de mensajes.

Usá la **TUI** (`apx code`) para:
- Sesiones de codificación interactivas donde querés salida en streaming, visibilidad de herramientas y cambio
  de modelo.
- Trabajo multi-turno extendido donde la transcripción completa es útil.

Usá el **REPL** (`apx chat`) para:
- Ida y vuelta con un agente específico del proyecto.
- Conversaciones que querés retomar más tarde por id.
- Situaciones donde Bun no está disponible o se prefiere una interfaz mínima.

<Aside type="tip">
  Los tres modos guardan sus conversaciones en `~/.apx/`. Ejecutá `apx sessions list` o
  `apx session find "<text>"` para ubicar trabajo pasado en todos los motores y modos.
</Aside>
